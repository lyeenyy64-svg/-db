## 인증 및 권한

### Slack 인증
- **로그인 화면**: "BAROGO DEBTFLOW" 로고 + "Slack으로 로그인" 버튼 (Slack 아이콘 포함)
- **인증 흐름**: Slack OAuth → 사용자 정보(이름, 이메일, Slack ID) 획득 → 자동 등록
- **신규 사용자**: 첫 로그인 시 자동 등록되나 **관리자 승인 필요** (승인 전까지 대기 화면 표시)
- **승인 대기 화면**: 아바타, 이름, 이메일 표시 + "관리자 승인 대기 중입니다" 안내 + 로그아웃 버튼
- **로그아웃**: 사이드바 하단에서 가능

### 사용자 권한 (Role)

| 권한 | 키 | 데이터 조회 | 추가/수정 | 삭제 | 어드민 접근 |
|------|-----|:---------:|:---------:|:----:|:----------:|
| 관리자 | `admin` | O | O | O | O |
| 편집자 | `editor` | O | O | O | X |
| 조회자 | `viewer` | O | X | X | X |

- 관리자만 어드민 메뉴가 사이드바에 표시됨
- 편집/삭제 버튼은 해당 권한이 없으면 숨김 처리
- 권한별 숨김 대상: [신규 등록], [수정], [삭제], [입금 등록], [활동 기록], [+추가] 버튼

## 1. 대시보드

### KPI 카드 (4개)
- 총 관리 채권 — 건수, 추심진행/보류 건수
- 총 채권 원금 — 원금 합계, 법무 기준 잔액
- 회수율 — 퍼센트, 총 회수액
- 법적절차 진행 — 압류 + 회생파산 건수

### 브랜드별 채권 현황
- 바로고(B) / 딜버(D) / 모아라인(M) 각각의 건수, 잔액, 막대 그래프

### 분류별 현황
- 장기채권 / 회생/파산 / 추심의뢰 건수
- 담당자별 현황

### 2026년 월별 회수실적
- 1~12월 막대 차트 (만원 단위)

### 요약 바
- 분할상환 진행 건수
- 채권압류 진행 건수
- 회생/파산 건수
- 긴급 알림 건수
- 총 입금건수

## 2. 채권관리

## 7. 어드민 (admin 권한만 접근 가능)

어드민은 **4개의 상위 탭**으로 구분된다. 탭 상태는 설정 변경 시에도 유지된다 (state를 상위로 분리).

### 7-1. 시스템 설정

코드값, 담당자, 브랜드 등 시스템 설정을 관리한다.

| 설정 카테고리 | config 키 | 설명 |
|---------------|-----------|------|
| 담당자 | `assignees` | 추심 담당자 목록 (추가/수정/삭제, 삭제 안전장치) |
| 브랜드 | `brands` | 브랜드 코드/이름/색상 (추가/수정/삭제, 삭제 안전장치) |
| 허브/지점 | `hubNames` | 허브/지점 목록 (추가/수정/삭제) |
| 채무발생원인 | `debtCauses` | 채무 발생 원인 코드 (추가/수정/삭제) |
| 추심상태 | `collStatuses` | 추심 상태 코드 (추가/수정/삭제) |
| 분류 | `categories` | 채권 분류 코드 (추가/수정/삭제) |
| 활동유형 | `activityTypes` | 추심 활동 유형 (추가/수정/삭제) |
| 입금채널 | `paymentChannels` | 입금 채널 코드 (추가/수정/삭제) |
| 납부시기 | `installmentTimings` | 분할상환 납부 시기 (추가/수정/삭제) |
| 법원 | `courts` | 관할 법원 목록 (추가/수정/삭제) |
| 죄명 | `chargeTypes` | 형사 죄명 코드 (추가/수정/삭제) |
| 경찰서 | `policeStations` | 경찰서 목록 (추가/수정/삭제) |

#### 각 카테고리 공통 기능
- 목록 표시
- 새 항목 추가 (입력 + 추가 버튼, Enter 키 지원)
- **항목 인라인 수정** (편집 아이콘 클릭 → 입력 필드 전환 → 저장/취소, Enter/Escape 지원)
- 항목 삭제 (확인 후)

#### 브랜드 전용 기능
- 코드 (1~2글자), 브랜드명, 색상 (컬러피커) 입력/수정
- **브랜드 인라인 수정** (코드/이름/색상 모두 수정 가능, 저장 시 채무자 데이터 연쇄 갱신)
- 색상 미리보기

### 7-2. 사용자 관리

Slack으로 인증된 사용자를 관리한다.

| 기능 | 설명 |
|------|------|
| 사용자 목록 | 아바타, 이름, 이메일, Slack ID, 가입일, 권한, 승인 상태, "나" 표시 |
| **승인/비활성화** | 신규 사용자 승인 버튼 또는 기존 사용자 비활성화 버튼 (자기 자신은 비활성화 불가) |
| **권한 변경** | 관리자/편집자/조회자 드롭다운 변경 (즉시 반영) |
| 권한 안내 | 각 권한별 접근 범위 설명 표시 |

### 7-3. 알림 설정

Slack 채널 또는 개인 DM으로 알림을 보내는 규칙을 관리한다.

#### 알림 규칙 구성
| 필드 | 설명 |
|------|------|
| 규칙 이름 | 알림 규칙 식별 이름 |
| 활성/비활성 | 토글 스위치 |
| 트리거 | 알림을 발생시키는 이벤트 유형 (드롭다운) |
| 조건 설명 | 상세 조건 텍스트 (예: "잔액 1,000만원 초과") |
| 알림 대상 | Slack 채널 또는 개인 DM (드롭다운) |
| 채널/대상자 | #채널명 입력 또는 승인된 사용자 DM 선택 (드롭다운) |

#### 트리거 유형
| 트리거 키 | 설명 |
|-----------|------|
| `installment_overdue` | 분할상환 미납 |
| `rehab_overdue` | 회생 변제금 미납 |
| `high_balance` | 고액 잔액 (1,000만원 초과) |
| `new_payment` | 신규 입금 등록 |
| `no_contact` | 장기 미연락 (30일) |
| `status_change` | 추심상태 변경 |
| `new_debtor` | 신규 채권 등록 |
| `seizure_collected` | 압류 회수 발생 |

#### 기능
- 규칙 추가 (+버튼) / 삭제 (휴지통)
- 인라인 편집 (펼침/접힘 토글)
- 활성/비활성 토글 스위치
- Slack 채널 또는 승인된 사용자 DM 선택
- 트리거 유형 안내 표 표시

#### 기본 등록 규칙 (5개)
| 규칙 | 트리거 | 대상 | 기본 상태 |
|------|--------|------|-----------|

### 7-4. 변경 로그

모든 데이터 추가/수정/삭제 이력을 조회한다.

#### 로그 항목 구성
| 필드 | 설명 |
|------|------|
| 시간 | 변경 일시 (YYYY.MM.DD HH:MM:SS) |
| **사용자** | **변경을 수행한 로그인 사용자명** (보라색 배지) |
| 작업 | 등록(초록) / 수정(파랑) / 삭제(빨강) 색상 배지 |
| 대상 | 채권 / 입금 / 추심활동 (회색 배지) |
| 상세 | 대상 이름, ID 등 요약 텍스트 |
| **변경 내역** (수정 시) | **항목별 기존값 → 변경값** 펼침 표시 |

#### 수정 로그 상세
- 채권 수정 시 변경된 필드만 추출하여 `기존값 → 변경값` 형태로 표시
- 기존값: ~~취소선~~ 빨간 배경 / 변경값: 초록 배경으로 시각 구분
- 비교 대상 필드 (20개): 브랜드, 분류, 담당, 채무자명, 연락처, 코드, 허브/지점, 채무발생원인, 추심상태, 원금잔액, 조정액, 회수액, 집행권원, 대여일자, 대위변제월, 신용조회일자, 신용등급, 주민등록초본, 영업담당자, 주요사항

#### 기능
- CSV 다운로드 (시간, 사용자, 작업, 대상, 상세, 변경항목, 기존값, 변경값 — 8개 컬럼)
- 최대 500건 보관 (FIFO)

## 데이터 연동 규칙

앱 내 데이터 변경 시 자동으로 연쇄 적용되는 규칙:

### 입금 등록 시
| 영향 대상 | 변경 내용 |
|-----------|-----------|
| `debtors.collectedAmount` | 입금액만큼 증가 |
| `debtors.finalBalanceFinance` | 원금잔액 - 회수액으로 재계산 |
| `debtors.finalBalanceLegal` | 최종잔액(재무) + 조정액으로 재계산 |
| `debtors.monthlyCollected[월]` | 입금월에 금액 합산 |
| `debtors.collectionStatus` | 잔액 0 이하 시 → "추심보류" 자동 변경 |
| `installmentPlans.logs[해당월].status` | 미납/지연 → "완납" 자동 변경 |
| `auditLogs` | 입금 등록 로그 기록 (사용자 포함) |

### 입금 삭제 시
| 영향 대상 | 변경 내용 |
|-----------|-----------|
| `debtors.collectedAmount` | 입금액만큼 감소 (원복) |
| `debtors.finalBalanceFinance` | 재계산 |
| `debtors.finalBalanceLegal` | 재계산 |
| `debtors.monthlyCollected[월]` | 해당월 금액 차감 |
| `auditLogs` | 입금 삭제 로그 기록 (사용자 포함) |

### 채무자 수정 시
| 영향 대상 | 변경 내용 |
|-----------|-----------|
| `payments.debtorName/brand` | 이름/브랜드 변경 시 연쇄 갱신 |
| `activities.debtorName/brand` | 이름/브랜드 변경 시 연쇄 갱신 |
| `seizureCases.debtorName/brand` | 이름/브랜드 변경 시 연쇄 갱신 |
| `rehabilitations.debtorName/brand` | 이름/브랜드 변경 시 연쇄 갱신 |
| `installmentPlans.debtorName/brand` | 이름/브랜드 변경 시 연쇄 갱신 |
| `complaints.debtorName/brand` | 이름/브랜드 변경 시 연쇄 갱신 |
| `auditLogs` | 변경 전/후 diff 포함 로그 기록 (사용자 포함) |

### 채무자 삭제 시
| 영향 대상 | 변경 내용 |
|-----------|-----------|
